# Анализ результатов тестирования алгоритмов сортировки строк

## 1. Merge Sort

### Теоретическая сложность

Сложность Merge Sort — O(n log n)

Так как сортируются строки, дополнительно учитываются посимвольные сравнения, поэтому итоговую сложность можно оценить как O(D log n), где D — суммарная длина строк

### Анализ графиков

Merge Sort работает стабильно на всех типах данных

Время работы растёт плавно вместе с размером массива

Разница между случайными, обратно отсортированными и почти отсортированными массивами небольшая

На графике сравнений видно, что количество сравнений постепенно увеличивается вместе с размером массива

На случайных и почти отсортированных данных сравнений больше, чем на обратно отсортированных

В целом Merge Sort показывает предсказуемую работу, но по скорости уступает специализированным алгоритмам

---

## 2. Quick Sort

### Теоретическая сложность

Средняя сложность Quick Sort — O(n log n)

В худшем случае сложность может ухудшаться до O(n²)

### Анализ графиков

На случайных данных Quick Sort работает быстро

На обратно отсортированных и почти отсортированных массивах время начинает расти заметно быстрее

Особенно это видно на больших размерах массивов

Это связано с неудачным разбиением массива при выборе опорного элемента

На графике сравнений видно такое же поведение

На обратно отсортированных данных количество сравнений становится очень большим

На случайных данных рост сравнений остаётся более плавным

Quick Sort сильно зависит от структуры входных данных

---

## 3. MSD Radix Sort

### Теоретическая сложность

MSD Radix Sort работает через посимвольную обработку строк

Теоретическая сложность — O(D + nR), где:

- D — суммарная длина строк
- R — размер алфавита

В работе использовался алфавит из 74 символов

### Анализ графиков

MSD Radix Sort показывает стабильное время работы

Рост времени остаётся плавным почти на всех тестах

Порядок элементов в массиве почти не влияет на производительность

На графике сравнений значения почти равны нулю

Это связано с тем, что алгоритм почти не использует обычные сравнения строк, а работает напрямую с символами

---

## 4. MSD Radix Sort + Quick Sort

### Теоретическая сложность

Алгоритм объединяет MSD Radix Sort и Quick Sort

Для больших подмассивов используется MSD Radix Sort, а для маленьких — Quick Sort

### Анализ графиков

Этот алгоритм показал лучшие результаты по времени работы

Даже на больших массивах время остаётся небольшим

На обратно отсортированных данных время выше, чем на случайных, но всё равно меньше, чем у обычного Quick Sort

Количество сравнений растёт более плавно, чем у Quick Sort

Алгоритм работает быстрее и стабильнее обычного Quick Sort

---

# Сравнение алгоритмов

Merge Sort работает стабильно, но уступает специализированным строковым алгоритмам по скорости

Quick Sort хорошо работает на случайных данных, но хуже — на отсортированных массивах

MSD Radix Sort показывает стабильную производительность и почти не зависит от порядка строк

Лучшие результаты показал MSD Radix Sort + Quick Sort

---

# Итоговые выводы

1. Результаты соответствуют теоретическим оценкам сложности алгоритмов

2. Merge Sort работает стабильно, но медленнее специализированных алгоритмов

3. Quick Sort сильно зависит от структуры входных данных

4. MSD Radix Sort обеспечивает стабильную производительность благодаря поразрядной обработке строк

5. MSD Radix Sort + Quick Sort показал лучшие результаты среди всех протестированных алгоритмов
