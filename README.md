# Анализ результатов тестирования алгоритмов сортировки строк

## 1. Merge Sort

### Теоретическая сложность

Для Merge Sort сложность составляет O(n log n) независимо от входных данных

Так как сортируются строки, дополнительно учитываются посимвольные сравнения, поэтому фактически сложность можно оценить как O(D log n), где D — суммарная длина строк

### Анализ графиков

По графикам видно, что время работы растёт достаточно плавно и предсказуемо

Алгоритм показывает стабильное поведение на всех типах данных

На случайных и почти отсортированных массивах время немного выше, чем на обратно отсортированных

На некоторых размерах массивов видны небольшие скачки времени, особенно около 1500–2200 элементов, но общий рост остаётся равномерным

График сравнений тоже выглядит достаточно ровным

Количество посимвольных сравнений постепенно увеличивается вместе с размером массива

Больше всего сравнений получилось на случайных и почти отсортированных данных

На обратно отсортированных массивах сравнений заметно меньше

Это связано с тем, что строки начинают различаться раньше, и алгоритму приходится сравнивать меньше символов

В целом результаты соответствуют теории и показывают стабильную работу Merge Sort

---

## 2. Quick Sort

### Теоретическая сложность

Средняя сложность Quick Sort — O(n log n)

В худшем случае сложность может ухудшаться до O(n²)

Для строк использовалась тернарная версия Quick Sort, которая лучше работает с одинаковыми префиксами

### Анализ графиков

На случайных данных Quick Sort показывает хорошие результаты и работает быстрее Merge Sort

Но на обратно отсортированных массивах время начинает расти заметно быстрее

Особенно сильный рост виден после 2000–2500 элементов

На почти отсортированных данных тоже появляются резкие скачки времени

Это связано с тем, что алгоритм начинает получать неудачные разбиения массива при выборе опорного элемента

График сравнений подтверждает это поведение

На обратно отсортированных данных количество сравнений растёт очень быстро и становится самым большим среди всех тестов

На почти отсортированных массивах тоже видно резкое увеличение сравнений ближе к большим размерам массивов

На случайных данных рост сравнений остаётся более плавным

В целом графики хорошо показывают зависимость Quick Sort от структуры входных данных

---

## 3. MSD Radix Sort

### Теоретическая сложность

MSD Radix Sort работает не через полное сравнение строк, а через посимвольную обработку

Теоретическая сложность алгоритма — O(D + nR), где:

- D — суммарная длина строк
- R — размер алфавита

В данной работе размер алфавита равен 74 символам

### Анализ графиков

По времени работы алгоритм показывает достаточно стабильное поведение

Рост времени остаётся плавным почти на всех тестах

На некоторых размерах массивов видны резкие скачки, особенно около 1400–1600 элементов

Скорее всего это связано с особенностями распределения строк по корзинам и рекурсивными вызовами

В целом производительность почти не зависит от степени отсортированности массива

На графике сравнений значения остались почти нулевыми

Это связано с тем, что MSD Radix Sort практически не использует обычные лексикографические сравнения строк

Алгоритм в основном работает через обработку символов и распределение строк по группам

Поэтому количество сравнений у него значительно меньше, чем у Merge Sort или Quick Sort

---

## 4. MSD Radix Sort + Quick Sort

### Теоретическая сложность

Этот алгоритм сочетает MSD Radix Sort и Quick Sort

Для больших подмассивов используется MSD Radix Sort, а для маленьких — Quick Sort

Такой подход уменьшает накладные расходы рекурсии и ускоряет сортировку

### Анализ графиков

Этот алгоритм показал лучшие результаты по времени работы

Даже на больших массивах время остаётся небольшим

Рост времени получается достаточно плавным

На обратно отсортированных данных время выше, чем на случайных, но всё равно заметно меньше, чем у обычного Quick Sort

На графике сравнений видно, что количество операций растёт более равномерно, чем у Quick Sort

На случайных данных сравнений меньше всего

На обратно отсортированных данных сравнений больше, но рост остаётся стабильным без резких скачков

Это показывает, что комбинация MSD Radix Sort и Quick Sort работает устойчивее обычного Quick Sort

---

# Сравнение алгоритмов

Merge Sort показывает стабильную работу на любых входных данных, но проигрывает специализированным строковым алгоритмам по скорости

Quick Sort хорошо работает на случайных данных, но сильно зависит от структуры массива

На отсортированных и почти отсортированных массивах его производительность заметно ухудшается

MSD Radix Sort показывает хорошую стабильность и почти не зависит от порядка строк в массиве

Лучшие результаты показал MSD Radix Sort + Quick Sort

Этот алгоритм оказался самым быстрым и при этом сохранял достаточно стабильное поведение на всех типах данных

# Итоговые выводы

1. Полученные результаты в целом соответствуют теоретическим оценкам сложности алгоритмов

2. Merge Sort работает стабильно, но уступает специализированным строковым алгоритмам по скорости

3. Quick Sort показывает хорошие результаты только на случайных данных и сильно зависит от структуры массива

4. MSD Radix Sort работает стабильно благодаря поразрядной обработке строк

5. MSD Radix Sort + Quick Sort показал лучшие результаты по времени работы и оказался самым эффективным алгоритмом среди протестированных

6. Специализированные алгоритмы сортировки строк позволяют уменьшить время работы и сократить количество операций по сравнению с обычными алгоритмами сортировки
